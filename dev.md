# 上证指数尾盘数据、板块与龙头股排行榜网页开发文档

## 一、项目目标

开发一个网页系统，用于采集、统计和展示上证指数相关行情、板块数据、板块内股票数据以及板块龙头股数据。

系统重点关注：

1. 上证指数实时表现
2. 板块成交额、成交量、资金量排行
3. 每个板块内股票排行
4. 板块龙头股排行
5. 小时榜、上午榜、下午榜、尾盘榜
6. 周六、周日、节假日不刷新，显示最近一个交易日数据

本项目先以 **AKShare 作为数据采集源** 快速实现 MVP。AKShare 官方文档中，`stock_zh_index_spot_em` 可获取沪深京指数实时行情，`stock_zh_a_spot_em` 可获取沪深京 A 股实时行情，板块与资金流相关接口也可作为排行榜数据来源。

---

## 三、数据源设计

### 1. MVP 阶段数据源

项目初期建议使用：

| 数据类型     | 数据源                              | 说明                         |
| -------- | -------------------------------- | -------------------------- |
| 上证指数     | AKShare `stock_zh_index_spot_em` | 获取上证指数点位、涨跌幅、成交量、成交额       |
| A 股实时行情  | AKShare `stock_zh_a_spot_em`     | 获取股票价格、涨跌幅、成交量、成交额、最高价、最低价 |
| 沪市 A 股行情 | AKShare `stock_sh_a_spot_em`     | 如果只关注上证相关股票，可优先使用沪市数据      |
| 板块行情     | AKShare 板块相关接口                   | 获取行业板块、概念板块行情              |
| 板块资金流    | AKShare 资金流相关接口                  | 获取行业、概念板块资金流排行             |
| 个股资金流    | AKShare 个股资金流相关接口                | 获取个股主力净流入、超大单、大单等数据        |
| 交易日历     | Tushare `trade_cal`              | 判断今天是否交易日，获取上一个交易日         |

Tushare 的 `trade_cal` 接口包含 `is_open` 字段用于判断是否交易，`pretrade_date` 字段用于获取上一个交易日，适合处理周六、周日、节假日不刷新数据的需求。

## 六、排行榜计算规则

AKShare 多数实时行情接口返回的是“当前累计成交量、累计成交额”等快照数据。小时榜、上午榜、下午榜、尾盘榜不能只看当前值，需要通过不同时间点的快照差值计算。

### 1. 成交量计算

```text
区间成交量 = 区间结束快照累计成交量 - 区间开始快照累计成交量
```

例如：

```text
14:30 - 15:00 尾盘成交量 = 15:00 累计成交量 - 14:30 累计成交量
```

---

### 2. 成交额计算

```text
区间成交额 = 区间结束快照累计成交额 - 区间开始快照累计成交额
```

---

### 3. 涨跌幅计算

```text
区间涨跌幅 = (区间结束价格 - 区间开始价格) / 区间开始价格 * 100%
```

---

### 4. 资金量计算

资金量建议优先使用：

```text
主力净流入-净额
```

如果接口字段不足或不稳定，可以先使用下面指标替代：

```text
资金量 = 成交额
```

后续再升级为：

```text
资金量 = 主力净流入净额
资金强度 = 主力净流入净额 / 成交额
```

注意：

资金流接口可能和实时行情接口刷新频率不一致，所以系统需要允许资金流数据延迟，并在页面展示资金数据更新时间。

## 七、龙头股定义规则

数据源一般不会直接告诉你某个板块“真正龙头股”是谁，所以系统需要自己定义。

建议支持两种方式：

---

### 1. 自动识别龙头股

每个板块内按以下优先级判断：

1. 成交额排名靠前
2. 主力净流入排名靠前
3. 涨跌幅排名靠前
4. 成交量排名靠前

推荐公式：

```text
龙头分数 =
成交额排名分 * 0.4
+ 主力净流入排名分 * 0.3
+ 涨跌幅排名分 * 0.2
+ 成交量排名分 * 0.1
```

每个板块取综合分最高的股票作为自动龙头股。

---

### 2. 手动配置龙头股

后台支持手动配置龙头股。

例如：

```text
板块：半导体设备
龙头股：北方华创、中微公司、拓荆科技
```

建议：

MVP 阶段先用自动识别。

正式使用时，增加手动配置功能。

---

## 八、交易日和非交易日规则

系统需要判断当前日期是否交易日。

### 1. 交易日

如果 Tushare `trade_cal` 返回：

```text
is_open = 1
```

则视为交易日。

---

### 2. 非交易日

如果出现以下情况，不刷新实时行情：

1. 周六
2. 周日
3. 法定节假日
4. 交易所休市日

如果 Tushare `trade_cal` 返回：

```text
is_open = 0
```

页面展示最近一个交易日数据。

最近一个交易日可以使用：

```text
pretrade_date
```

---

## 九、数据刷新规则

### 1. 交易时间内

建议刷新频率：

| 数据      | 刷新频率       |
| ------- | ---------- |
| 上证指数    | 5 - 15 秒   |
| A 股实时行情 | 15 - 30 秒  |
| 板块行情    | 30 - 60 秒  |
| 资金流数据   | 60 - 180 秒 |
| 排行榜缓存   | 30 - 60 秒  |

---

### 2. 午休时间

午休时间：

```text
11:30 - 13:00
```

处理方式：

* 不重复高频采集
* 页面显示上午最后一条数据
* 下午开盘后继续刷新

---

### 3. 收盘后

收盘后：

```text
15:00 之后
```

处理方式：

* 保存当天最终快照
* 生成当日总榜
* 生成尾盘榜
* 页面展示当天最终数据

---

### 4. 周六、周日、节假日

处理方式：

* 不刷新实时数据
* 展示最近一个交易日数据
* 页面顶部提示非交易日
* 排行榜展示最近一个交易日最终榜单

## 十三、开发建议

### 1. 不建议前端直接取第三方数据

原因：

* 容易跨域
* 不方便限频
* 不方便缓存
* 不方便统一清洗字段
* 不方便计算小时榜、尾盘榜
* 容易暴露接口细节

正确结构：

```text
AKShare / Tushare
        ↓
Python 后端采集
        ↓
PostgreSQL 存快照
        ↓
Redis 缓存排行榜
        ↓
FastAPI 提供接口
        ↓
前端展示
```

---

### 2. 数据采集要做适配层

不要在业务代码里到处直接调用 AKShare。

建议封装：

```text
DataProvider
    ├── AkshareIndexProvider
    ├── AkshareStockProvider
    ├── AkshareSectorProvider
    ├── AkshareFundFlowProvider
    └── TushareCalendarProvider
```

这样后续换 Choice、Wind、iFinD 或上证所 Level-1 数据时，不需要重写业务逻辑。

---

### 3. 所有排行榜从数据库快照计算

不要直接拿实时接口结果排序后展示。

正确做法：

1. 定时采集快照
2. 入库
3. 根据快照差值计算时间段数据
4. 生成排行榜
5. 前端读取排行榜接口

---

### 4. MVP 先做这些功能

第一版只做：

1. 上证指数展示
2. 板块成交额榜
3. 板块成交量榜
4. 板块涨幅榜
5. 龙头股成交额榜
6. 龙头股成交量榜
7. 上午榜
8. 下午榜
9. 尾盘榜
10. 非交易日显示最近交易日数据

资金流可以第二版再增强。

## 十六、重要注意事项

1. AKShare 接口字段可能变化，所以必须做字段校验。
2. 行情接口可能偶尔失败，所以必须做重试。
3. 资金流数据可能不是秒级实时，所以页面需要展示资金数据更新时间。
4. 排行榜必须从快照计算，不能只拿当前接口结果。
5. 非交易日必须显示最近一个交易日数据。
6. 不要让前端直接依赖第三方行情接口。
7. 后期商业化必须考虑行情数据授权。
8. 龙头股最好支持手动配置，不要完全依赖自动算法。
9. 所有采集任务必须写日志。
10. 所有 API 返回结构要统一，方便前端维护。

---

## 十七、最终目标

最终网页需要让用户快速看到：

1. 当前上证指数表现
2. 哪些板块成交额最高
3. 哪些板块成交量最大
4. 哪些板块资金最活跃
5. 哪些板块尾盘异动明显
6. 哪些龙头股成交量放大
7. 哪些龙头股资金量靠前
8. 上午、下午、小时、尾盘不同阶段的市场强弱变化
9. 非交易日自动展示最近一个交易日最终行情
